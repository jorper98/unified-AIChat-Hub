# Plan: Add Date/Time and Weather Awareness to Kilo

## Problem Summary

1. **Date/Time**: When the user asks "what day/time is today", the AI doesn't know. The system actually injects `Current time` into `environment_details` (visible in system reminders), but the AI may not reliably use it or it may get pruned during context condensing.

2. **Weather**: The AI has no built-in weather capability. It needs an external data source.

## Solutions

### 1. Current Date/Time

**Option A (Recommended): Fix via Custom Instructions**
The `environment_details` already contains `Current time: 2026-06-03T18:34:59-07:00`. Add an instruction to `AGENTS.md` or a custom rule file telling the AI to reference this value:

- Create/edit `AGENTS.md` (project-level) or `~/.config/kilo/AGENTS.md` (global) with:
  ```
  When the user asks about the current date, time, or day of week, check the "Current time" value in the environment_details section at the top of the system prompt.
  ```

**Option B: Use the `bash` tool (built-in)**
The AI already has access to `bash`. If instructed properly, it can run `date` or `Get-Date` (PowerShell) to get the current time. Add to instructions:
  ```
  When asked about the current date/time, run the `date` command (or `Get-Date` on Windows) to get the exact current time.
  ```

**Option C: Add dynamic time via a command template**
Create a `/now` command in `.kilo/command/now.md` that uses the shell output template variable `!`cmd`` to inject the current date/time into the prompt.

### 2. Current Weather

**Recommended: MCP Weather Server**
Configure a weather MCP server in `kilo.json` (project or global). This gives the AI a dedicated tool to fetch weather data.

Steps:
1. Install a weather MCP server (e.g., `@modelcontextprotocol/server-weather` or a community alternative)
2. Add to `kilo.json` under the `mcp` key:
   ```jsonc
   {
     "mcp": {
       "weather": {
         "type": "local",
         "command": ["npx", "-y", "@modelcontextprotocol/server-weather"],
         "environment": {
           "OPENWEATHER_API_KEY": "your-api-key"
         },
         "enabled": true
       }
     }
   }
   ```
3. Set permission to auto-approve weather reads:
   ```jsonc
   {
     "permission": {
       "weather_*": "allow"
     }
   }
   ```

**Alternative: Shell-based weather command**
Create a `.kilo/command/weather.md` that fetches weather via a shell command (e.g., `curl wttr.in` or a weather API) and injects the result.

### 3. Skills and Memory (General)

**Skills** (extend AI capabilities):
- Create `skills/<name>/SKILL.md` in `.kilo/skills/`, `~/.config/kilo/skills/`, or custom paths
- Frontmatter requires `name` and `description`
- Add custom skill directories in `kilo.json`:
  ```jsonc
  {
    "skills": {
      "paths": ["./my-skills", "~/shared-skills"],
      "urls": ["https://example.com/.well-known/skills/"]
    }
  }
  ```

**Memory / Persistent Context**:
- `AGENTS.md` at project root or `~/.config/kilo/AGENTS.md` for global instructions
- Additional instruction files via `instructions` array in `kilo.json`:
  ```jsonc
  {
    "instructions": [
      ".kilo/rules/*.md",
      "./docs/coding-standards.md"
    ]
  }
  ```
- Global instructions: `~/.config/kilo/AGENTS.md`
- Per-directory instructions: place `AGENTS.md` in any subdirectory

**Commands** (reusable prompt templates):
- Create `.kilo/command/<name>.md` with YAML frontmatter
- Use `$1`-`$N` for positional args, `!`cmd`` for shell output, `@file` for file contents

## Implementation Steps

### Step 1: Date/Time Awareness
Add instruction to `~/.config/kilo/AGENTS.md` (create if not exists):
```markdown
## Current Date/Time
When the user asks about the current date, time, or day of week, check the "Current time" value in the environment_details section of the system prompt. If not available, run the `date` command (Windows: `Get-Date`) to get the exact current time.
```

### Step 2: Weather MCP Setup
Add to `~/.config/kilo/kilo.json` (or `kilo.jsonc`):
```jsonc
{
  "mcp": {
    "weather": {
      "type": "local",
      "command": ["npx", "-y", "@anthropic/mcp-server-weather"],
      "enabled": true,
      "timeout": 10000
    }
  },
  "permission": {
    "weather_get_current_weather": "allow",
    "weather_get_forecast": "allow"
  }
}
```
**Note**: User will need an `OPENWEATHER_API_KEY` environment variable set.

### Step 3: Verify Configuration
Check that both files exist and are valid JSON/Markdown.

## Prerequisites
- User needs an OpenWeather API key (free at https://openweathermap.org/api)
- User should set it as an environment variable: `OPENWEATHER_API_KEY`

## Tradeoffs

| Approach | Pros | Cons |
|----------|------|------|
| AGENTS.md instruction | Simple, no dependencies | Static text, AI may still miss it |
| Bash tool instruction | Always accurate | Requires AI to execute a command |
| MCP Weather | Clean, dedicated tool | Requires API key, external dependency |
| Shell command weather | No API key needed (wttr.in) | Less reliable, rate-limited |

## User Decisions

- **Scope**: Global configuration (all projects) via `~/.config/kilo/`
- **Weather**: MCP Server approach (requires OpenWeather API key from user)
